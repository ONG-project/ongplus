### “Como usuário, quero explorar causas, conhecer ONGs e acompanhar a transparência das ações para realizar doações de forma consciente e segura.”

### Critérios de aceitação

O usuário deve conseguir visualizar informações da ONG.

O usuário deve visualizar metas e arrecadações.

O usuário deve conseguir verificar evidências de confiabilidade.

O usuário deve visualizar histórico de campanhas.

### Regras funcionais

O histórico de arrecadações deve permanecer acessível.

Informações financeiras devem ser exibidas de forma transparente.

Relatórios públicos devem possuir data de atualização.

### Regras de negócio

O doador deve conseguir filtrar ONGs por causa.

O doador deve conseguir visualizar histórico de campanhas.

O doador deve conseguir ver status de verificação.

O doador deve conseguir verificar se as informações foram alteradas desde a última verificação.

### Regras de interface

Deve ser possível ver histórico de campanhas.

Deve ser possível ver status de verificação.

Deve ser possível solicitar alterações via formulário.

Deve ser possível visualizar feedback sobre alterações.
